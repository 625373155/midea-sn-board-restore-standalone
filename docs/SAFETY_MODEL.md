# 安全模型

## 两阶段边界

源代码工具与设备运行包是两个边界：

1. `wizard/generate/validate/self-test/history` 只处理本地文件，不导入网络客户端，不连接设备。
2. 生成包由用户核对后手动运行；它绑定一个 SN、一个服务热点、一个型号和一个事件 ID。

生成器永远不替用户跨越第二阶段。

## 资格门

生成写入能力前必须同时成立：所有权/授权、型号精确为已验证的 `KFR-26G/WXAA2@` 美的空调、真实新实体主板、可信完整原 SN、当前实时热点、准确型号/换板证据，以及无同一事件写入保留。热点格式相同不能扩展型号兼容范围。

SN 单独不构成资格。零响应、配网失败或热点存在也不能证明板为空。

## 历史与锁

生成器在包可见之前，将 `PACKAGE_GENERATION_RESERVED` 追加到 Windows LocalAppData 已知文件夹中的 JSONL 历史。相同 SN 或热点再次出现时，默认拒绝；只有后续真实换了另一块实体板，才可链接上一事件并建立新事件。

新板凭据保存原文用于人工核对，但防重复比较使用 `nfkc-casefold-whitespace-v1` 哈希：NFKC 规范化、Unicode casefold、连续空白折叠后再做 SHA-256。大小写、全角/兼容字符或空白变化不能伪造新的事件证据。

运行包在打开写入连接前建立全局和包侧写入保留。保留后发生崩溃、超时、部分发送、无 ACK 或 0 字节，都不能重试。

锁建立后、`NetworkStream.Write` 尚未开始时发生的异常归类为 `WRITE_NOT_SENT_BUT_LOCKED`：审计记录会明确说明本进程未开始网络写，但锁不会被删除，也不会自动恢复写资格。

这些锁不是防篡改硬件，无法阻止能修改源码或删除文件的人。

## 证据强度

从强到弱：完整性校验并解码为精确目标 SN 的协议读回；经冷启动、型号展示关系已确认且功能正常的官方 App；完整 ACK 或非目标读回；原始字节数/连接/超时/零字节。

“完整性通过”不等于身份有效。身份字段还必须用逆置换和逆替换表严格解码为恰好 22 位 ASCII 数字。无法解码的完整回复归类为 `READ_ONLY_INVALID_ENCODING`，同时写入包内和全局 `READ-INVALID-DO-NOT-WRITE` 永久停止标记，不能降级成普通不匹配或空码。

TCP 连接和 ACK 不证明写入成功。零字节不证明失败。结果不明确时必须停止写入，只做只读或官方售后检查。
