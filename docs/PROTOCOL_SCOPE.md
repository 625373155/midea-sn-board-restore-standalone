# 协议范围摘要

本文件帮助代码审计，不是通用设备协议说明。

- 产品类型固定为 `0xAC`（空调）。
- 已验证型号 allowlist 固定为 `KFR-26G/WXAA2@`。
- 目标固定为当前服务 WLAN 上的 `192.168.1.1:6444`。
- 身份固定为生成时核验的一个 22 位 ASCII 数字 body SN。
- 读操作为 `0x40`，写操作为 `0x41`。
- 内层帧固定 37 字节，并校验 CRC-8 与二补数校验和。
- 传输层对完整内层帧执行 AES-128-ECB 和 PKCS#7 填充；本协议的该消息传输帧为 104 字节。
- TCP 按字节流解析；必须检查头、长度、填充、产品类型、CRC 和校验和，再通过严格逆编码得到 22 位 ASCII 数字后，才能比较 SN。完整性通过但不可解码的载荷触发永久 `READ-INVALID-DO-NOT-WRITE` 停止标记。
- 只读路径结构上不能接收写操作；写操作只有一个静态发送点，且位于所有验证重试循环之外。

协议可执行基准位于 `midea_sn_restore/protocol.py`，公开回归数据位于 `midea_sn_restore/test_vectors.json`，只含合成身份。任何实现差异都应视为发布阻断问题。
