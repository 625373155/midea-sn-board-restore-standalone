# 安全与隐私策略

## 支持范围

仅支持当前 `main` 分支和最新发布版本。该项目只面向设备所有者或获授权维修人的美的空调实体换板后原机身 SN 恢复。

## 私下报告安全问题

请优先使用公开仓库 [625373155/midea-sn-board-restore-standalone](https://github.com/625373155/midea-sn-board-restore-standalone) 的 **Security → Report a vulnerability** 私密报告功能。若该功能尚未启用，请只提交不含设备身份的最小公开问题描述，并请求维护者提供私密沟通渠道。

报告中不得包含真实完整 SN、SSID、BSSID、`TARGET.json`、生成 ZIP、App 截图、客服材料、未经脱敏的日志或令牌。GitHub Personal Access Token 等密钥一旦出现在聊天、日志或 Issue 中，应立即撤销并重新创建。

## 公开数据规则

仓库和测试只能使用明确标记的合成身份。Pull Request 必须通过隐私扫描，且不得提交：

- 真实设备身份或维修事件；
- 本机 `%LOCALAPPDATA%\MideaSnBoardRestore` 历史；
- 生成包、写入标记、读回标记或诊断日志；
- 任意账号、云端绑定信息或售后凭据。

发布前必须运行 `py -3 .\public_release_check.py`。扫描范围包括候选工作树文件的内容、路径和文件名、NUL/非 UTF-8 内容、符号链接、`IMMUTABLE_PRIOR_EVENTS` 空字面量，以及所有 refs 可达的提交树、提交消息、ref 名和标签消息。只扫描当前文件不足以清除旧提交或旧标签中的隐私数据。

如误提交，单纯删除最新文件不够；必须先撤销相关密钥、阻止仓库公开，并清理所有可达 Git 历史与标签后再发布。

## 安全不变量

- 离线生成器不得导入网络客户端或连接设备。
- 产品类型固定为空调 `0xAC`，运行端点固定为 `192.168.1.1:6444`。
- SN 必须是可信来源的精确 22 位 ASCII 数字；SN 单独不足以生成。
- 服务 SSID 必须是实时观察到的精确 `midea_test_<12 hex>`。
- 兼容型号 allowlist 固定为 `KFR-26G/WXAA2@`；热点格式不能证明其他型号兼容。
- 只读路径不得构造或发送 `0x41`。
- 完整性通过的 22 字节身份载荷必须严格解码后才允许比较；无法解码时必须同时建立包内和全局 `READ-INVALID-DO-NOT-WRITE` 永久标记。
- 新板凭据的防重复哈希必须使用版本化的 NFKC、casefold 和空白折叠规范化；仅改变大小写、兼容字符或空白不得绕过历史门禁。
- 写入保留先于网络发送建立；一次事件最多一个写请求，没有 ACK 或 0 字节都不能重试。
- 写锁建立后到 `NetworkStream.Write` 开始前的任何异常必须追加 `WRITE_NOT_SENT_BUT_LOCKED`，且不得自动解锁或重试。
- 模板、哈希、ZIP 或静态检查失败时必须停止。
- 禁止批量、克隆、云账号绕过、隐藏开关和任意目标主机。

任何削弱上述不变量的变更都属于发布阻断问题。

## 威胁模型边界

本地标记和历史是尽力而为的软件防护，不是防篡改硬件。能编辑源代码、复制文件到另一台电脑或控制用户文件系统的人可绕过本地状态。项目依靠技术门槛与真实所有权/换板证据共同降低误用，不能保证阻止恶意修改者。

工具不处理云端账号，不存储远程凭据，也不声称可以修复强电、制冷剂或硬件故障。
